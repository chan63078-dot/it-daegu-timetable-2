## 3. API 설계

### Base URL: `http://localhost:5000`

| Method | Endpoint | 설명 | 파라미터 | 응답 |
|--------|----------|------|----------|------|
| `GET` | `/` | 메인 HTML 페이지 | - | HTML |
| `GET` | `/api/timetable` | 시간표 전체 데이터 | `?month=4&type=weekday` | JSON (grid + courses) |
| `GET` | `/api/courses` | 과정 목록 (검색용) | `?q=검색어&month=4&type=weekday` | JSON |
| `GET` | `/api/files` | 사용 가능한 파일 목록 | - | JSON |

**파라미터 값**
- `type`: `weekday` (평일) / `weekend` (주말)
- `month`: `4` ~ `8`

## 4. 핵심 파싱 로직 (`csv_service.py`)

```
1. CSV 파일을 cp949 인코딩으로 읽기

2. 행 0~1에서 강의실 컬럼 맵핑 생성
   - 행 0: 그룹명 (A[10층], B[10층] ...)  → 헤더 색상 그룹 지정용
   - 행 1: 세부 강의실명 (A[10층]-1, A[10층]-2 ...)  → 실제 컬럼 키

3. 행 2에서 강의실별 정원 / 수강인원 파싱
   - 패턴: "26???[ 재: 20]" → 정원 26, 수강인원 20

4. 행 3~끝: 시간 블록 순회
   - 열 0 값이 'H:MM' 또는 'HH:MM' 패턴이면 새 시간 블록 시작
   - 이전 블록 텍스트 수집 완료 → 강의실별 과정 객체 생성
   - 각 강의실 열의 비어있지 않은 텍스트를 누적

5. 각 텍스트 행 분류 규칙 (우선순위 순):
   - '전체출석율' / '당일출석율' 포함  → 출석율 필드
   - '정원 :' 패턴                    → 정원 / 수강인원
   - '배정:' 패턴                     → 배정 인원
   - '개:YYYY-MM-DD' 패턴            → 개강일
   - '종:YYYY-MM-DD' 패턴            → 종강일
   - '수업없음'                       → 수업 없음 플래그
   - '월~금' / '토/일' / '월수금' 등 → 요일
   - '휴강:' / '보강:' 포함          → 비고 (일정 특이사항)
   - '★' 포함                        → 비고 (위탁강사 정보)
   - 날짜 목록 패턴 ('M/D,D,...')    → 비고 (수업일 목록)
   - 나머지 첫 번째 텍스트           → 과정명
   - 나머지 두 번째 텍스트           → 강사명

6. rowspan 계산
   - 동일 강의실에서 연속된 시간 블록에 같은 과정명이 있으면
     첫 블록의 rowspan을 증가, 이후 블록은 병합 처리로 표시 생략
```

## 6. 기술 결정 사항 (ADR)

### ADR-01: 데이터 소스 — CSV 읽기 전용
- **결정:** 파일 수정 기능 없이 읽기 전용 뷰어로 구현
- **이유:** 요구사항이 조회·검색에 집중됨, 원본 파일 보호
- **트레이드오프:** 데이터 수정이 필요하면 원본 CSV 직접 편집 필요

### ADR-02: 파싱 방식 — 블록 기반 순차 파싱
- **결정:** 시간 행을 기준으로 블록을 나누고 각 열의 텍스트를 규칙 기반으로 분류
- **이유:** CSV가 시간대 × 강의실 그리드 구조이므로 행 단위가 아닌 블록 단위 파싱 필요
- **트레이드오프:** 규칙에 맞지 않는 예외 데이터는 비고 필드로 fallback

### ADR-03: 프론트엔드 — Vanilla JS (프레임워크 없음)
- **결정:** React/Vue 없이 순수 JS로 구현
- **이유:** 번들러 없이 CDN 활용, 배포 단순화
- **트레이드오프:** 상태 관리 직접 구현 필요
